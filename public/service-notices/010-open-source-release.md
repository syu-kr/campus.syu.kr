안녕하세요, SYU CAMPUS입니다.

SYU CAMPUS의 소스 코드를 GitHub에 공개했습니다. 서비스가 어떻게 만들어지고 운영되는지 살펴보고, 학습과 연구, 버그 리포트, 기능 개선 제안 등 다양한 목적으로 활용해 주세요.

---

## 공개 저장소

- [SYU CAMPUS GitHub 저장소](https://github.com/syu-kr/campus.syu.kr)

공개된 코드는 Next.js, React, TypeScript, Tailwind CSS를 기반으로 구성되어 있으며, 공지사항·학식·셔틀버스·학사일정 등 서비스 기능의 구현과 운영 자동화 코드를 확인할 수 있습니다.

코드는 MIT 라이선스에 따라 이용할 수 있습니다. 복사, 수정, 배포 등 구체적인 이용 조건은 저장소의 `LICENSE` 파일을 반드시 확인해 주세요. 코드와 별개로 로고, 상표, 학교 및 외부 기관에서 제공된 데이터에는 별도의 권리가 적용될 수 있습니다.

---

## 이렇게 활용해 주세요

공개 저장소를 다음과 같은 방식으로 활용하는 것을 환영합니다.

- 웹 서비스 구조와 구현 방식 학습
- 개인 프로젝트 및 연구의 참고 자료
- 버그 발견 및 재현 방법 제보
- 사용성 개선과 새로운 기능 제안
- 문서, 접근성, 성능 개선
- 직접 수정한 코드의 Pull Request 제출

단순한 오탈자 수정부터 기능 개선까지 다양한 참여를 기다리고 있습니다.

---

## 코드 기여 전 반드시 확인해 주세요

코드를 수정하여 기여하려면 저장소의 [`CONTRIBUTING.md`](https://github.com/syu-kr/campus.syu.kr/blob/main/CONTRIBUTING.md)와 [`DEVELOPMENT.md`](https://github.com/syu-kr/campus.syu.kr/blob/main/DEVELOPMENT.md)를 먼저 읽고 안내된 방식을 정확히 따라야 합니다.

주요 기여 절차는 다음과 같습니다.

1. 기능과 관련된 변경을 작은 단위로 작성합니다.
2. 운영 데이터에 dummy 또는 mock 데이터를 추가하지 않습니다.
3. 환경 변수가 추가되면 예시 파일과 관련 문서를 함께 수정합니다.
4. 제출 전 `npm run check`를 실행하여 lint, 타입 검사, 미사용 코드, 크롤러 문법, 프로덕션 빌드를 확인합니다.
5. 사용자 동작이나 배포 방식이 바뀌면 관련 문서를 함께 수정합니다.
6. 변경 사항은 Pull Request로 제출하고 필수 검사가 통과될 때까지 확인합니다.

기여 가이드를 따르지 않거나 운영 안정성에 영향을 줄 수 있는 변경은 반영되지 않을 수 있습니다.

## 버그와 보안 취약점 제보

일반적인 버그와 개선 의견은 GitHub Issue 또는 사이트 문의를 통해 알려주세요. 재현 절차, 발생 환경, 예상 동작과 실제 동작을 함께 작성하면 더 빠르게 확인할 수 있습니다.

보안 취약점은 공개 Issue에 상세 내용, 비밀 값, 개인정보를 올리지 마세요. 저장소의 **Security > Report a vulnerability** 기능을 이용해 비공개로 제보해야 합니다. 해당 메뉴를 사용할 수 없다면 세부 정보를 공개하지 않은 상태로 `singhic_dev@syu.kr`에 먼저 연락해 주세요.

자세한 내용은 저장소의 [`SECURITY.md`](https://github.com/syu-kr/campus.syu.kr/blob/main/SECURITY.md)를 확인해 주세요.

## 공개 코드를 이용한 공격 행위 금지

소스 코드 공개는 운영 중인 SYU CAMPUS 서비스나 연결된 학교·외부 시스템에 대한 공격, 침투 테스트 또는 무단 접근을 허가한다는 의미가 아닙니다.

다음과 같은 행위는 엄격히 금지됩니다.

- 운영 서비스의 정상적인 기능을 방해하거나 서버에 의도적으로 부하를 주는 행위
- 공개 코드를 분석하여 인증 우회, 무단 접근, 해킹 또는 시스템 침해를 시도하는 행위
- 취약점을 악용하거나 타인의 개인정보 및 비공개 정보를 수집하는 행위
- 악의적인 크롤링, 스크래핑, 자동화 도구 또는 봇 사용
- 학교 시스템이나 외부 연동 서비스에 피해를 주는 행위

이러한 행위는 SYU CAMPUS [이용약관](/terms)에 따라 서비스 이용 제한 대상이 되며, 행위의 내용과 피해에 따라 관계 법령에 의한 형사처벌 및 민사상 손해배상 책임의 대상이 될 수 있습니다. 테스트와 연구는 반드시 본인이 통제하는 로컬 환경 또는 적법하게 허가받은 환경에서만 진행해 주세요.
